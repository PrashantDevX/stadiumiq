# Security Policy

## Supported version

Security fixes are applied to the current `main` branch and the latest deployed
version of StadiumIQ.

## Reporting a vulnerability

Please do **not** open a public issue for a suspected vulnerability. Report it
privately to the repository owner through GitHub's private vulnerability
reporting feature, or email the maintainer listed in the repository profile.

Include a clear description, affected route or component, reproducible steps,
and any proof of concept. You should receive an acknowledgement within seven
days. We will investigate, coordinate a fix, and credit reporters who want
recognition after a patch is available.

## Handling secrets

Never include API keys, service-account JSON, or production incident data in a
report. If a secret is exposed, revoke or rotate it immediately and then report
the exposure through a private channel.
